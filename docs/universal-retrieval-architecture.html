Universal Document Ingestion, Research, and Report Creation

An agent framework that transforms your document library into a queryable knowledge base — enabling analysts to retrieve information, synthesise research across sources, and produce structured reports through a conversational chat interface.

Unified Storage
Persisted in three Postgres tables for catalog, content, and tabular retrieval
PostgreSQL + pgvector
iris_document_catalog
One entry per document. Stores summary, usage description, and metadata. Used by the planner to identify and select relevant sources.
iris_document_content
One entry per page or sheet. Stores extracted text, keywords, and vector embeddings. Retrieved via hybrid semantic + keyword search during research.
iris_document_tables
One entry per detected table. Small tables returned in full; large tables loaded as typed rows and filtered via SQL. Used by the agent when numerical or tabular lookups are needed.
